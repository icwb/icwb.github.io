  • Apa itu Indonesian Civil War Bot 2030/ICWB?

    Indonesian Civil War Bot 2030 atau singkatnya ICWB adalah page bot sejenis WorldWarBot 2020 dimana bot ini otomatis memposting simulasi invasi dari suatu daerah ke daerah lain secara acak.

  • Istilah-istilah yang digunakan

    - Botmin: Bot Admin/Admin Page/Developer Bot/yang buat bot/yang punya page bot. Case disini yaitu Admin page ICWB.
    - Run: Sesi simulasi yang sedang atau sudah berlangsung, dari awal bot dijalankan sampai akhir.
    - Bot: Program yang menjalankan beberapa tugas secara otomatis.

  • "Min apakah ICWB beneran bot?"

    Postingan yang tidak diawali dengan [Botmin post] diposting oleh bot. Botmin juga bisa memposting tanpa diawali dengan [Botmin post].

  • "Ini game bukan sih min? Cara mainnya gimana?"

    Ini bukan game yang bisa kalian mainkan. Dan sepertinya page ini tidak bisa disebut game.

  • "Bagaimana sih cara kerja bot ini? Apakah cara kerjanya sama kayak WWBot 2020?"

    Botmin tidak akan menjelaskan secara detail alur yang dipakai oleh bot, tetapi dibawah ini adalah cara kerja bot secara singkat:

    • Daerah pertama dipilih secara acak
    • Daerah kedua dipilih secara acak dari daerah yang berdekatan dengan daerah pertama
    • Penyerang ditentukan dari pemilik daerah pertama
    • Lawan ditentukan dari pemilik daerah kedua
    • Penyerang menguasai daerah kedua dari Lawan
    • Ada kondisi dimana suatu provinsi yang telah kalah dapat mendeklarasikan kemerdekaan
    • Ada juga kondisi dimana penyerang kemungkinan gagal menguasai daerah kedua
    • Kondisi diulangi sampai satu provinsi tersisa

    Bisa dibandingkan dengan WWBot 2020

  • "Apakah project botnya open-sourced?"

    ICWB menggunakan bahasa pemrograman Typescript/Javascript (NodeJS). Boleh difork, boleh diclone, boleh dipelajari atau bahkan dimodifikasi jika ingin membuat page seperti ini. (tidak ada dokumentasi untuk source code, jika ingin memodifikasi, good luck).

    Note: Pull Request tidak akan ditanggapi.

    Suatu provinsi masih dapat menguasai daerah manapun jika provinsi tersebut masih menguasai setidaknya satu daerah.

Changelog v2:

Di season kedua, source code bot yang dipakai untuk menjalankan ICWB berbeda dengan season pertama, tetapi alur yang dipakai tidak jauh berbeda. Botmin merefaktor hampir keseluruhan source code yang dipakai untuk season 2. Ada beberapa penambahan dan optimisasi dari source code v2 yang akan membuat run season kedua lebih cepat berakhir dari pada season pertama. Akibat dari refaktor dan optimisasi ini adalah setiap run yang berlangsung kemungkinan tidak bisa lebih dari 150 giliran (atau mungkin tidak akan melebihi tahun 2045 dalam satu run), dan ping-pong menjadi cukup jarang terjadi. Botmin masih mencoba untuk membuat suatu run cukup panjang dan tidak terlalu membosankan (contohnya membuat dua provinsi tidak saling berebut daerah berturut-turut selama beberapa giliran, atau bisa disebut kedua provinsi tersebut melakukan ping-pong).

Inilah daftar perubahan untuk source code v2:

  • [Add] - Failing system. Sistem yang memungkinkan sebuah penyerangan akan gagal. Kemungkinannya adalah 0.3 atau 30% suatu penyerangan akan gagal. Ini dibuat untuk sedikit memperpanjang sebuah run.
  • [Update] - Independence system. Provinsi yang baru merdeka hanya dapat dikuasai setelah provinsi diserang sebanyak 3 kali. Probabilitas diubah menjadi 0.4 atau 40% dan berkurang setengahnya setiap ada provinsi yang merdeka. Sistem ini akan terpicu jika jumlah provinsi yang tersisa lebih dari 5 dan kurang dari 20.
  • [Update] - Map sekarang menggunakan data dari gadm.org.
  • [Update] - Warna setiap provinsi telah diperbaharui. Kemungkinan akan berubah setiap run dilakukan.
